## Purpose
Dashboards provide real‑time overviews of project health using Dataview queries.

## How They Work
Each dashboard queries notes with specific frontmatter types and displays aggregated results.

## Setup Checklist
- [ ] Install Dataview plugin in Obsidian
- [ ] Create at least one note of each type (milestone, risk, problem) using the templates
- [ ] Open each dashboard to verify queries work
- [ ] Customize queries as needed

## Troubleshooting
- If tables appear empty, check that your notes have the correct `type` field.
- Ensure notes are inside the `ATOM‑SG Pilot` vault (or adjust the `FROM` path).
- Verify frontmatter syntax (YAML with `---` delimiters).
